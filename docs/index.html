Install pjm-api

Install the package locally. Use the `pfx` extra so certificate inspection works.

python -m pip install -e ".[pfx]"

Certificate model

PJM uses two certificate files. The most common setup mistake is mixing them up.

Login file

Use locally: `.p12` or `.pfx` with private key and certificate.

Give this path to `pjm-api init`. Do not commit it.

Public certificate

Upload to PJM: `.cer` or `.crt` without private key.

Account Manager and CAM approval make browserless login possible.

What doctor checks

`pjm-api doctor` prints the first thing to fix. `--offline` skips PJM network calls but still checks local setup.

Turn setup into scripts

After `doctor` passes, user scripts can import the same pieces the CLI uses. Keep secrets in `pjm-api init`; do not hard-code them.

Small script

from pjm_api import CliBackend, load_settings
from pjm_api.templates import suggest_params

backend = CliBackend(load_settings())
result = backend.run_template(
    template="TRANSSERV",
    params=suggest_params("TRANSSERV"),
    outfile="transserv.txt",
)
print(result.returncode, result.output_file)

Beginner questions

Why does this project use Java?

PJM provides an official Java CLI for browserless OASIS access. `pjm-api` makes that path easier to configure, test, and call from Python.

Do I need to export `PJM_CLI_JAR_PATH`?

Not if the jar is installed at `~/.pjm/cli/pjm-cli.jar`. That default path is auto-detected. Use the environment variable only for a different location.

Where should I start when something fails?

Run `pjm-api doctor`. Fix the first failed line, then run it again.

Can I use production?

Start in TRAIN. Production reads warn by default, and production write or reservation-style actions are blocked unless explicitly enabled.

pjm-api is unofficial and is not affiliated with PJM.
